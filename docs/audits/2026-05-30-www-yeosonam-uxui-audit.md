# www.yeosonam.com UX/UI 개선 발굴 감사

- 감사일: 2026-05-30 KST
- 대상: `https://www.yeosonam.com` 공개 사이트맵 233개 URL, 주요 고객 템플릿 7종 데스크톱/모바일, 주요 CTA 22회 클릭
- 산출물:
  - `docs/audits/2026-05-30-live-domain-page-audit.json`
  - `docs/audits/2026-05-30-live-domain-customer-cta-audit.json`
  - `docs/audits/2026-05-30-live-ui-screens/`

## 결론

Figma/MCP 연동은 이번 단계의 필수 조건이 아니다. 실제 라이브 UX/UI 문제는 Playwright 기반 브라우저 감사만으로 충분히 드러났다. Figma MCP는 "목표 디자인 시안과 실제 구현의 차이"를 픽셀/토큰 단위로 비교할 때 유용하고, Vercel/Supabase 연동은 500 오류와 RLS/로그인을 포함한 운영 문제를 추적할 때 보조적으로 쓰면 된다.

가장 먼저 잡을 P0는 모바일 컨시어지 레이아웃 붕괴, 목적지 상세 500, 전 페이지 CSP 오류, 패키지 상세의 비로그인 `user_actions` 401/RLS 실패다. 디자인 개선으로 보이는 문제 중 상당수가 실제 전환 흐름과 신뢰도에 영향을 준다.

## 범위와 한계

공개 사이트맵 기준으로 감사했다. `/admin`, `/tenant`, 개인 예약 링크, 토큰 기반 공유 링크처럼 인증/권한/실데이터가 필요한 화면은 별도 로그인 세션 또는 테스트 계정으로 Chrome/Playwright 인증 감사가 필요하다.

## P0: 즉시 수정

### 1. 모바일 AI 컨시어지 레이아웃 붕괴

- 증상: 모바일 `/concierge`에서 좌측 대화 영역이 극단적으로 좁아져 `어디로 떠나실까요?`가 글자 단위로 세로 배치된다.
- 영향: 핵심 AI 상담 진입점이 모바일에서 사실상 사용 불가.
- 근거: `docs/audits/2026-05-30-live-ui-screens/mobile-concierge.png`
- 관련 파일: `src/app/concierge/page.tsx`
- 개선안: 모바일에서는 장바구니 사이드바를 숨기거나 하단 시트/토글 패널로 전환한다. 메인 대화 영역은 `min-width: 0`, `width: 100%`를 보장하고, 데스크톱에서만 2열 레이아웃을 적용한다.

### 2. 목적지 상세 일부 500

- 증상: 사이트맵과 CTA 감사에서 `/destinations/다낭`, `/destinations/계림%2F양삭` 등이 500을 반환한다.
- 영향: 목적지 허브에서 카드/탭 클릭 시 실패 요청이 발생하고, SEO 색인 페이지가 깨진다.
- 근거: 사이트맵 감사 233개 중 하드 오류 12개, CTA 실패 3건.
- 관련 파일: `src/app/destinations/[city]/page.tsx`
- 개선안: city slug 정규화 규칙을 통일한다. 쉼표, 슬래시, 공백 포함 도시명은 canonical slug와 display name을 분리하고, 잘못된 slug는 301 또는 안전한 not-found로 처리한다.

### 3. 전 페이지 CSP 오류

- 증상: 233개 전 페이지에서 `connect-src`의 `https://o*.sentry.io`가 invalid source로 무시된다.
- 영향: 콘솔 신뢰도 하락, Sentry 이벤트 누락 가능성, 실제 오류 탐지 방해.
- 개선안: CSP의 Sentry source 패턴을 유효한 호스트 와일드카드로 교정한다. 예: `https://*.sentry.io` 또는 실제 DSN host.

### 4. 패키지 상세 비로그인 액션 추적 401/RLS 실패

- 증상: 다수 `/packages/[id]`에서 Supabase `user_actions` 401 및 RLS 실패가 반복된다.
- 영향: 고객에게 보이지 않더라도 성능/로그 품질/분석 정확도에 악영향. 콘솔 오류가 모든 상품 상세의 기본 상태가 된다.
- 개선안: 익명 추적 이벤트는 서버 API를 통해 익명 허용 정책으로 저장하거나, 클라이언트 직접 insert를 중단한다. 실패는 콘솔 error가 아니라 샘플링된 warn/telemetry로 낮춘다.

## P1: 전환과 신뢰도 개선

### 5. 홈/패키지 목록 H1 부재

- 증상: `/`, `/packages`에 시각적 페이지 제목은 있지만 실제 `h1`이 없다.
- 영향: 접근성, SEO, 스크린리더 탐색 품질 저하.
- 관련 파일: `src/app/page.tsx`, `src/app/packages/page.tsx`, `src/app/packages/PackagesClient.tsx`
- 개선안: 첫 화면의 주 메시지를 `h1`로 승격하되 기존 디자인은 유지한다. 홈은 "믿고 떠나는 프리미엄 패키지 여행", 패키지는 "해외 패키지 검색"처럼 검색 의도와 맞춘다.

### 6. 모바일 하단 고정 CTA와 채팅 버튼 충돌

- 증상: 홈, 패키지 목록, 목적지, 견적 요청, 상품 상세에서 보라색 채팅 플로팅 버튼과 하단 탭/예약 바가 겹친다.
- 영향: 클릭 실수, CTA 위계 혼란, 상품 상세 예약 버튼 방해.
- 개선안: 모바일에서는 플로팅 채팅 버튼 위치를 하단 네비 높이와 예약 고정바 높이에 따라 동적으로 올린다. 상품 상세에서는 하단 예약바 안에 카톡 CTA를 흡수하는 편이 낫다.

### 7. 패키지 목록 이미지 로딩 실패/플레이스홀더

- 증상: `/packages` 첫 카드와 모바일 목록에 지구본 플레이스홀더가 반복된다.
- 영향: 여행 상품의 신뢰와 탐색 욕구를 떨어뜨린다. 특히 목록 첫 화면에서 치명적이다.
- 개선안: 대표 이미지가 없을 때 지역별 고품질 fallback 이미지를 제공하고, 이미지 없는 상품은 목록 랭킹에서 후순위로 내린다.

### 8. 단체 견적 요청 화면의 빈 공간과 입력 위계

- 증상: `/group-inquiry`는 데스크톱/모바일 모두 첫 대화 이후 화면 대부분이 비어 있고, 입력창이 하단에 떨어져 있다. 모바일에서는 안내 말풍선 일부가 상단 고정 영역에 가려진다.
- 영향: 사용자가 "무엇을 해야 하는지"는 알지만 즉시 입력할 동기가 약하다.
- 개선안: 첫 화면에 빠른 선택 칩을 목적지/인원/일정 예시로 배치하고, 모바일 sticky header 높이를 줄이거나 메시지 영역의 top padding을 보정한다.

### 9. AI 컨시어지 데스크톱 첫 화면 밀도 부족

- 증상: `/concierge` 데스크톱은 검색창 아래 대부분이 빈 공간이고 장바구니가 우측을 크게 차지한다.
- 영향: AI가 무엇을 할 수 있는지 체감하기 전에 화면이 비어 보인다.
- 개선안: 검색 전 상태에는 추천 요청 템플릿, 최근 인기 조합, 결과 예시 skeleton을 제공한다. 장바구니는 상품이 없을 때 폭을 줄이거나 접는다.

### 10. 블로그 에디터 픽 카드 가독성 저하

- 증상: `/blog` 에디터 픽의 배경 로고/큰 텍스트와 제목이 겹쳐 시각적 소음이 크다.
- 영향: 콘텐츠 신뢰도와 읽기 유도가 떨어진다.
- 개선안: 카드 안의 장식 텍스트 대비를 낮추고 제목/메타/CTA만 분명히 남긴다. 우측 작은 카드와 같은 정보 구조로 정렬한다.

## P2: 시스템화하면 좋은 개선

### 11. 공개 페이지별 CTA 명칭 통일

홈은 "단체·맞춤 견적", 글로벌 내비는 "단체 문의", 하단 탭은 "단독맞춤"처럼 같은 흐름의 명칭이 조금씩 다르다. 사용자는 같은 기능인지 다른 기능인지 헷갈릴 수 있다. 고객 여정 기준으로 "단체 맞춤 견적" 또는 "맞춤 상담" 중 하나를 정해 통일한다.

### 12. 카드 랭킹 숫자의 의미 보강

홈 인기 패키지 카드의 큰 숫자 1, 2, 3은 시선을 잘 끌지만 가격/지역/상품명보다 강하다. 랭킹 badge는 유지하되 이미지 좌상단 작은 badge로 낮추고 상품명과 가격 위계를 올리는 편이 전환에 유리하다.

### 13. 목적지 카드 이미지 부재

`/destinations` 카드가 모두 같은 gradient와 지구본 아이콘으로 보여 목적지 차이가 약하다. 최소한 지역별 대표 사진 또는 색상/랜드마크 패턴을 다르게 적용한다.

### 14. 필터 영역의 모바일 높이 과다

`/packages` 모바일 첫 화면은 검색/필터가 화면 절반 가까이를 차지한다. 필수 필터만 접힌 상태로 두고, 정렬/지역 칩은 결과 상단 sticky row로 유지한다.

### 15. 접근성 기본값

홈/패키지 H1 부재 외에도 버튼/아이콘 CTA가 emoji 의존도가 높다. 시각 장식 emoji는 `aria-hidden` 처리하고, 아이콘 버튼에는 명확한 `aria-label`을 부여한다.

## 추천 실행 순서

1. `/concierge` 모바일 레이아웃 수정
2. `/destinations/[city]` slug 500 수정
3. CSP 및 `user_actions` 추적 오류 제거
4. 홈/패키지 H1 보강
5. 모바일 플로팅 CTA 충돌 정리
6. 패키지/목적지 이미지 fallback 개선
7. `/group-inquiry`, `/concierge` 빈 상태 UX 보강

## 추가 연동 판단

- Figma MCP: 지금은 필수 아님. 디자인 원본이 있고 "시안 대비 구현 차이"를 잡을 때 연결.
- Chrome 플러그인: 로그인 세션이 필요한 관리자/테넌트/마이페이지 감사에 필요.
- Vercel 플러그인: `/destinations` 500의 production logs, deployment/env 차이 확인에 유용.
- Supabase 플러그인: `user_actions` RLS/익명 추적 정책 수정 시 필요.
- Lighthouse/PageSpeed: 다음 단계에서 LCP/CLS/접근성 점수화에 유용.
