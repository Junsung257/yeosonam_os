# 단독맞춤여행 생태계 개선 로드맵

> 작성일: 2026-05-26
> 목표: 견적 요청 → 랜드사 배정 → 견적 전달 → 공유 → 히스토리 락인의 완전한 루프 구축

---

## 현재 상태 요약

| 단계 | 상태 | 문제 |
|------|------|------|
| 고객 견적 제출 | ✅ 양호 | private-tour 4단계 폼 완료 |
| RFQ 저장 | ✅ 양호 | /api/rfq → group_rfqs |
| 운영자 알림 | ⚠️ Slack만 | 실시간 대시보드 없음 |
| 랜드사 배정 | ⚠️ 수동 | 운영자가 publish 필요 |
| 랜드사 입찰 | ✅ 존재 | bidding 흐름 있음 |
| 고객 제안서 비교 | ⚠️ B2B 느낌 | 텍스트 위주, 시각적 비교 부족 |
| 견적 공유 | ❌ 없음 | 일행과 공유 불가 |
| 히스토리/락인 | ❌ 없음 | 여행 이력 축적 안 됨 |
| 계약/결제 | ⚠️ 프로토타입 | 에스크로 미연동 |

## 개선안 7가지

### 1. 모바일 RFQ 전용 페이지 고도화 (`/rfq/[id]` 리뉴얼)

**현재 문제**: "단체여행 견적 현황"이라는 제목, B2B 느낌의 제안서 비교

**개선 방향**:

- **헤더**를 출발지/도착지/인원/유형을 담은 카드형으로 변경
  - 예: "✈️ 김해 → 다낭 | 가족여행 4명 | 2박3일"
  - 제안서를 여행사 로고+가격 카드 형태로 시각화 (지금은 A사/B사 → 실제 브랜드명 or '프리미엄/스탠다드/이코노미')
- **AI 추천 뱃지**: "이 여행사는 가족여행 120건 진행, 만족도 4.8"
- **진행 상태**를 프로그레스 바로 시각화 (지금도 있음, 개선 필요)
- 모바일 최적화: 하단 고정 CTA ("일행에게 공유하기", "이 제안 선택하기")

### 2. 견적 공유 시스템 (일행 초대)

**핵심 기능**: RFQ에 `share_token`을 생성하고, 일행이 링크로 접속하면 '읽기 전용 뷰어'로 보게 함

**구체적 동작**:

```
/private-tour 견적 제출
  → /rfq/[id] 로 리디렉션
    → "📤 일행에게 공유" 버튼
      → 공유 링크 생성: /share/rfq/[share_token]
        → 일행이 열면 '읽기 전용' 뷰 (브랜딩 + 가격 + 일정)
        → "👍 좋아요 / 🤔 궁금해요 / 💬 의견 남기기" 반응 가능
        → 투표 기능: "A안 vs B안 중 골라주세요"
```

**기술적 구현**:
- `group_rfqs`에 `share_token uuid DEFAULT gen_random_uuid()` 컬럼 추가
- `/share/rfq/[token]` 페이지 생성 (서버 컴포넌트, DB에서 share_token으로 조회)
- 읽기 전용 전용 페이지, 로그인 불필요
- 반응/투표는 익명 + 쿠키 기반 (중복 방지)

### 3. 여행 Passport (히스토리 락인)

**핵심 개념**: 여행을 다녀올 때마다 "Passport 스탬프"가 쌓이는 개인 여행 이력

**구체적 동작**:

```
마이페이지 → "나의 여행 Passport"
  → 여행 완료될 때마다 자동 추가
    → [🇻🇳 다낭 2026.06] [🇯🇵 오사카 2026.03] [🇹🇭 방콕 2025.12]
    → 각각 클릭하면 그때의 견적/일정/사진 리뷰
    → "이번에도 여소남?" → 재견적 버튼 (기존 정보 프리필)
```

**락인 효과**:
- 고객이 다음 여행을 계획할 때 과거 여행 이력을 보면서 자연스럽게 재방문
- "작년 다낭 때는 A랜드사 좋았는데, 이번 방콕도 A사로 해볼까?"
- 재견적 시 과거 정보 자동 프리필 → 전환율↑

**기술적 구현**:
- `customer_profiles` 테이블 (phone_number 기반 auto-merge)
- `user_travel_history` 테이블 (rfq_id → proposal_id → booking_id 연결)
- 마이페이지에 "여행 passport" 섹션
- passport 데이터를 기반으로 한 **재방문 쿠폰/혜택** 연동 가능

### 4. AI 요약 자동화 (랜드사 효율)

**현재 문제**: 운영자가 고객의 raw 요청을 보고 랜드사에 전달해야 함

**개선 방향**:

```
고객 제출 (custom_requirements JSONB)
  → AI가 자동으로 요약 생성
    → "📋 [AI 요약] 다낭 가족여행 4명, 2박3일, 1인 80~120만원
       - 65세 이상 어르신 동행 → 천천히 이동
       - 유아(2세) 동반 → 카시트 필요
       - 호텔 4성급 선호, 리조트형"
    → 이 요약이 RFQ 상단에 고정
    → 랜드사는 이 요약만 보고 입찰 결정
    → 운영자는 별도 전달 작업 불필요
```

### 5. 랜드사 입찰 대시보드 (운영자 효율)

**현재 문제**: 운영자가 새 RFQ를 보고, 수동으로 publish하고, 입찰 현황을 트래킹해야 함

**개선 방향**:

- `/m/admin/rfqs` 페이지 전용 생성 (지금은 bookings에 섞여 있음)
- 새 RFQ 도착 시 **푸시 알림** (PWA or Slack)
- RFQ publish 버튼 → "공개" 시 선택된 랜드사에 자동 알림 (Kakao/Solapi)
- 실시간 입찰 현황 (몇 개사 참여, 남은 시간)
- AI 자동 추천: "이 RFQ에 적합한 랜드사 TOP 3 추천" (과거履行 데이터 기반)

### 6. 전용 진행 링크 (모바일 랜딩)

**현재**: 고객이 제출 후 `/rfq/[id]`로 이동하지만, 이 페이지가 private-tour 전용 브랜딩이 아님

**개선 방향**:

- `/private-tour/trip/[share_token]` — **모바일 전용 랜딩 페이지**
- 여소남 브랜딩 + 여행 정보 + 제안서 카드 + 공유 버튼 + 실시간 진행
- PWA처럼 보이는 웹앱 형태
- URL 자체가 예뻐야 공유하기 좋음: `yeosonam.com/private-tour/trip/danang-2026`
- QR 코드 생성 (모바일에서 바로 열 수 있게)

### 7. 재견적 (1-Click Re-quote)

**락인 효과의 마무리**: 여행 passport에서 바로 이어짐

- Passport에서 과거 여행 선택 → "같은 스타일로 다시 견적받기"
- 과거 RFQ의 destination, pax, budget, hotel_grade를 그대로 프리필
- 새 여행이므로 날짜/인원만 수정
- 제출 시 "재방문 고객" 태그 자동 추가
